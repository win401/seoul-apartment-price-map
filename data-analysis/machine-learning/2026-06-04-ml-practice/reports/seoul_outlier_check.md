# Seoul Outlier Check

## Purpose

서울 아파트 실거래가 데이터에서 모델링 전에 확인해야 할 이상치 후보를 정리한다.

이상치는 무조건 삭제하지 않는다. 먼저 데이터의 의미를 보고, 모델 목적에 따라 유지/제외/분리 여부를 결정한다.

## Unit

- `transaction_real_price`: 만원 단위
- `transaction_price_eok`: 억 원 단위 보기용 컬럼
- `price_per_pyeong`: 만원/평 단위

## Summary

전체 서울 데이터: 742,285건

| 기준 | 건수 | 비율 |
| --- | ---: | ---: |
| 20억 초과 거래 | 4,733 | 0.638% |
| 평당 1억 초과 | 22 | 0.003% |
| 전용면적 10㎡ 미만 | 0 | 0.000% |
| 전용면적 250㎡ 초과 | 129 | 0.017% |
| 건물 나이 음수 | 3 | 0.000% |
| 층수 0 이하 | 144 | 0.019% |
| 층수 80 초과 | 0 | 0.000% |

## Interpretation Draft

- 20억 원 초과 거래는 단순 오류라기보다 고가 주택 시장일 수 있다.
- 평당 1억 원/평 초과 거래는 작은 면적, 특정 지역, 특수 거래, 데이터 오류 가능성을 함께 확인해야 한다.
- 건물 나이가 음수인 경우는 거래연도보다 준공연도가 미래라는 뜻이므로 오류 후보에 가깝다.
- 첫 번째 베이스라인 모델에서는 데이터를 과하게 삭제하지 않고, 이상치 포함 모델과 제한 모델을 비교하는 방향이 좋다.

## Preprocessing Draft

| 항목 | 기준 초안 | 처리 방향 |
| --- | --- | --- |
| 거래가 | 20억 원 초과 | 고가 시장으로 분리 검토 |
| 평당 단가 | 1억 원/평 초과 | 샘플 확인 후 유지/제외 결정 |
| 면적 | 10㎡ 미만 또는 250㎡ 초과 | 특수 거래/오류 확인 |
| 건물 나이 | 음수 | 오류 후보로 제외 검토 |
| 층수 | 0 이하 또는 80층 초과 | 데이터 정의 확인 |

## Next Action

1. 노트북에서 샘플을 직접 확인한다.
2. 베이스라인 모델용 데이터셋을 만든다.
3. 이상치 포함 모델과 기본 제한 모델을 비교한다.
