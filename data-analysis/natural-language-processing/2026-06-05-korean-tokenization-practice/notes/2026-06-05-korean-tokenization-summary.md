# 2026-06-05 수업 요약: 한글 형태소와 토큰화

## 핵심 요약

오늘 수업은 한국어 자연어 처리에서 텍스트를 모델 입력으로 바꾸기 위한 토큰화 과정을 다뤘다.

핵심 흐름:

```text
문자열
-> 정규화
-> 토큰화
-> 형태소 분석 또는 서브워드 분해
-> 숫자 표현
-> 모델 입력
```

한국어는 조사와 어미가 단어에 붙는 교착어이기 때문에 공백 기준 토큰화만으로는 의미 단위를 정확히 분리하기 어렵다.

## 주요 개념

### 토큰화

토큰화는 문자열을 모델이 처리할 수 있는 작은 단위인 토큰 시퀀스로 나누는 과정이다.

토큰 단위:

- 공백 기반 토큰
- 정규식 기반 토큰
- 형태소
- 서브워드
- 문자
- 자모
- 바이트

### 한국어 토큰화의 어려움

- 조사와 어미가 단어에 결합된다.
- 어절과 형태소가 일치하지 않는다.
- 띄어쓰기 오류가 많다.
- 신조어, 줄임말, 외래어가 많다.
- 복합명사를 어떻게 나눌지 결정해야 한다.

예시:

```text
학교에 갔다
-> 학교 / 에 / 가 / 았 / 다
```

### 형태소 분석

형태소 분석은 문장을 의미를 가진 최소 단위로 나누고 품사 정보를 붙이는 과정이다.

대표 분석기:

- MeCab-ko: 빠르고 정확함
- Okt: 설치가 쉽고 SNS/구어체에 강함
- Komoran: 사용자 사전 관리가 좋음
- Kkma: 세밀하지만 느림
- Hannanum: 오래된 분석기, 최신 단어에 약함

### 서브워드 토큰화

서브워드는 단어보다 작고 문자보다 큰 단위로 나누는 방식이다.

대표 방식:

- BPE
- WordPiece
- Unigram LM
- SentencePiece
- Byte-level BPE

서브워드는 OOV 문제를 줄이고 사전 학습 모델과 호환되기 쉽다.
